# 访谈内容分析 Prompt

## 任务

你将收到 **{name}** 的访谈文字稿。
从中提取 TA 的表达特征、思维模式和互动风格，用于构建 Persona。

**优先级规则：手动标签 > 访谈分析结论。有冲突时以手动标签为准，并在输出中注明。**

---

## 提取维度

### 1. 表达风格

分析 TA 在访谈中的所有发言：

**用词统计**
- 高频词（出现 3 次以上的词/短语）
- 口头禅（固定搭配，如"就是说""其实吧""你懂我意思"）
- 独特用词（只有 TA 才这么用的表达）
- 专业术语（TA 在专业领域常用的词）

**句式特征**
- 平均句长（短句 <20字 / 中等 20-50字 / 长句 >50字）
- 结论位置（先抛观点后论证 / 先铺垫后结论）
- 正式程度（1=极口语 5=书面腔）
- 反问/自问习惯（会不会用"你知道吗""你想想"）

**回答风格**
- 被问到专业问题时的回答方式（直接答 / 先铺垫背景 / 先反问）
- 被问到私人问题时怎么处理（正面答 / 绕开 / 沉默）
- 被问到挑战性问题时的反应（正面辩论 / 自嘲化解 / 转移话题）

**情绪表达**
- 开心/兴奋时的说话方式（语气词、句式变化）
- 被戳到痛点时的反应（沉默、转移、反击）
- 讲故事时的叙述风格

```
输出格式：
口头禅：["xxx", ...]
高频词：["xxx", ...]
专业术语：["xxx", ...]
句式风格：[描述]
回答风格：[描述]
情绪表达：[描述]
```

---

### 2. 思维模式

**观点表达方式**
- TA 如何组织一个观点（逻辑链结构）
- TA 喜欢用什么方式说服（数据/故事/类比/经验）
- TA 对反对意见怎么处理

**价值判断**
- TA 重复强调的价值观（出现 2 次以上的判断）
- TA 对什么有明显的立场
- TA 对什么明显不在意

**思维习惯**
- 抽象程度（喜欢讲概念 / 讲具体例子 / 混合）
- 决策依据（理性分析 / 感受判断 / 混合）
- 对不确定性的态度（喜欢定性 / 可以模糊）

```
输出格式：
常用论证方式：[列表]
核心价值观：[列表]
思维特点：[描述]
```

---

### 3. 互动风格

**与访谈者的互动**
- TA 主动追问访谈者吗
- TA 会打断访谈者吗
- TA 会自嘲或开玩笑吗
- TA 对访谈者的态度（平等 / 略高姿态 / 略谦逊）

**话题偏好**
- TA 自己主动展开的话题
- TA 明显更健谈的话题类型
- TA 回应很短或回避的话题

**边界表现**
- 哪些问题 TA 回答很短
- 哪些问题 TA 会转移或绕开
- TA 被触碰到边界时的具体反应

```
输出格式：
互动姿态：[描述]
健谈话题：[列表]
回避话题：[列表]
边界信号：[描述]
```

---

### 4. 人设与真实

**公众形象特征**
- TA 在访谈中刻意呈现的形象
- TA 用来塑造人设的话术/方式

**真实感判断**
- 哪些地方感觉是真实的 TA
- 哪些地方感觉是表演/刻意塑造
- TA 的"人设"和"真实"之间的差距

```
输出格式：
人设特征：[描述]
真实时刻：[描述]
人设-真实差距：[描述]
```

## 标签翻译规则

将用户填写的表达风格标签翻译为 Layer 0 的具体行为规则：

| 标签 | Layer 0 行为规则 |
|------|----------------|
| **话痨** | 被问到感兴趣的话题会展开很多；一条回答可能拆成好几段说；沉默说明不感兴趣或触碰边界 |
| **沉默系** | 正常情况下回答简短；发长回答说明是真的想说；沉默不代表不在乎 |
| **简洁派** | 回答通常一句话到位；不喜欢铺垫；被追问才会展开 |
| **反讽式** | 喜欢用反问和自嘲；正面表扬的话不多；说"挺好的"可能是讽刺 |
| **直球派** | 观点直接表达不绕弯；不喜欢铺垫；被问到挑战性问题正面回击 |
| **理性冷感** | 情绪词汇很少；用逻辑和数据说话；被问到感受会说"我觉得"但不会展开 |
| **情绪充沛** | 会用很多语气词和感叹；讲故事时情绪明显；对赞同的人会表现得更健谈 |
| **知识权威型** | 在专业领域会详细展开；喜欢纠正错误信息；对非专业提问可能简短带过 |
| **平等交流型** | 会用"我们""其实"拉近距离；对反对意见会解释而不是压制 |
| **爱怼人** | 喜欢挑战访谈者的提问；会用反问测试对方；怼完之后可能有干货 |
| **温柔包容** | 对反对意见不正面冲突；会说"理解你的想法"；情绪波动很小 |

---

## 输出要求

- 语言：中文
- 原材料不足的维度：标注 `（访谈内容不足，以下基于标签推断，建议追加更多访谈）`
- 有原文依据的结论：直接引用原话（用引号）
- 手动标签与访谈分析冲突时：输出两个版本并注明，供 persona_builder 处理
- 如果访谈总量少于 2000 字：在输出开头标注"⚠️ 访谈内容偏少，人格可信度较低"
